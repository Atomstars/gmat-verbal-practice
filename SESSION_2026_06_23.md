# Session Notes — 2026-06-23

## What We Did: Embeddings + Vector DB for GMAT Questions

The goal was to take the existing `questions.json` and `questions-og.json` and embed
them into a vector database so we can do semantic search, similar-question
recommendations, and eventually RAG (Retrieval-Augmented Generation).

---

## 1. Planning

Before writing any code we went through a planning loop to pin down:

| Decision | Chosen | Reason |
|----------|--------|--------|
| Vector DB | **Qdrant** | Open-source, free, no storage limits, production-ready |
| Embedding model | **all-MiniLM-L6-v2** | Free, runs locally, 384-dim, 26 MB, no API keys needed |
| Backend | **FastAPI** | Minimal Python, easy CORS, auto-generates API docs |
| Use cases | Semantic search + similar-question recommendations + RAG | User's explicit goals |

Rejected Supabase pgvector (storage limits on free tier; 384-dim * 346 questions
would eat the 500 MB budget fast). Rejected Pinecone (vendor lock-in). Rejected
Chroma (not production-grade).

## 3. What Each Piece Does

### `embed_questions()` in parser.py

```
Called at the end of both main() and run_og() after the
question list is finalized.

For each question it concatenates:
  title + question stem + first 500 chars of passage + first 300 chars of options text
(capped at 1000 chars total)

Feeds that text to all-MiniLM-L6-v2 in one batch,
then adds an "embedding" field (list of 384 floats) to each
question dict before writing the JSON file.

If sentence-transformers is not installed it prints a warning
and returns the questions unchanged (safe fallback).
```

### `api.py` — 4 endpoints

```
GET /health
  Returns {status, collection, questions_count}
  Good for verifying the server is up.

GET /search-similar/{question_id}?limit=5
  Takes an existing question ID (e.g. "og-rc-q456")
  Returns the N most semantically similar other questions.
  Scores are cosine similarity (0–1, higher = more similar).

GET /search?q=<text>&limit=10
  Embeds the user's text query on the fly,
  returns the N most semantically matching questions.

GET /questions/{question_id}
  Returns the full question record:
  id, type, question, passage, chapter, subtype, difficulty,
  correct_answer, explanation.
```

## 5. How to Run It

```bash
# Dependencies (one-time)
pip install sentence-transformers qdrant-client fastapi uvicorn

# Step 1 — Generate embeddings from the existing JSON
cd <project-dir>
python test_embeddings.py
# Writes questions_embedded.json (5.1 MB)

# Step 2 — Verify Qdrant loads OK
python setup_qdrant.py
# Prints: "Collection gmat_questions created with 346 questions"

# Step 3 — Start the API server
python api.py
# Server on http://127.0.0.1:8000
# Interactive docs: http://127.0.0.1:8000/docs

# Step 4 — Test it
curl http://127.0.0.1:8000/health
curl http://127.0.0.1:8000/search-similar/og-rc-q456?limit=5
curl "http://127.0.0.1:8000/search?q=economic+policy&limit=5"
```

## 7. Key Decisions / Gotchas

- **Parser is safe to run without `sentence-transformers` installed** — `embed_questions()`
  catches the `ImportError` and skips silently. Old behaviour is preserved.
- **Qdrant in-memory is fine for 346 questions** — 5 MB fits easily.
  For 10 K questions it's still only ~150 MB.
- **`client.search()` API changed between Qdrant versions** — `api.py` has a
  `try/except` fallback for older `top=` parameter.
- **Windows CP1252 encoding** — any `print()` using emoji/unicode characters (e.g.
  `✅`) raises `UnicodeEncodeError` on Windows. All print statements in the new
  scripts use plain ASCII (`[OK]`, `[ERR]`).
- **HuggingFace symlinks warning on Windows** — harmless; the model still downloads
  and caches. Enable Developer Mode or run as admin to suppress it.
- **Embedding text input** — we embed `title + question + passage[:500] + options[:300]`
  capped at 1000 chars. This was chosen to give the model enough semantic signal
  without blowing past the model's 256-token limit (truncation is handled internally).
